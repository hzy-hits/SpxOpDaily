# Order-map pricing backtest (2026-07-13)

## 结论

历史报告调用了 Black-Scholes 重定价，没有绕过 BSM。主要缺陷是把依赖未来触达时间的情景价格标成了可提前放置的静态期权限价。

短时间、近距离触价时模型表现可用；等待数小时的远墙误差明显扩大，不能输出单一可执行价格。

## 数据与样本

- 范围：2026-07-08 至 2026-07-13 的 systemd 15 分钟报告与 IBKR SPXW L1 lake。
- 从报告解析到 166 个合约/情景价格，135 个能与同到期 IBKR 合约行情配对。
- 仅当首次目标穿越时的 Greeks underlier 距目标不超过 2.5 点、bid/ask 为正且 live 时，才计入触价价格误差。
- 最终有效首次触价样本：59。

## 9.12 跳到 4.30 的原因

7500C 的报告价格由以下链路产生：

1. 以 IBKR 当前 mid 锚定模型，而不是直接采用纯 BSM 绝对价格。
2. 用 `0.6 * (distance / expected_move)^2` 估计触达耗时，最大可占剩余期限 90%。
3. 把剩余期限减去估计耗时。
4. 用本地 smile slope 调整触位 IV。
5. 计算目标位 BSM，并按 `market_mid * target_BS / anchor_BS` 比例重定价。

13:00 至 13:15，SPX 参考从约 7533 上移到 7539，7500 墙距离从约 33 点扩大到 39 点。距离超过约一个 expected move 后，耗时比例接近 90% 上限，触位剩余时间从数小时压缩到约 1.5 小时。因此价格从 9.12 降至 4.30。变化来自单一触达时间假设的非线性放大，不是 BSM 没有执行。

## 修复

- 所有候选统一为 `underlier_triggered_limit`，不再存在 `resting_limit` / `stop_trigger` 两套容易误读的语义。
- 报告改称“BS 触位情景价”和“触发后限价参考”。每张候选明确要求 SPX 触位后再按实时 mid/IV 重算。
- payload 新增 IV now/touch、tau now/touch、touch-time fraction、BS anchor/target 等完整复算字段。
- 每次 status/refresh/baseline 原始模板、结构化候选、LLM 输出和投递结果写入：
  `data_root/audit/order_map_pricing/date=YYYY-MM-DD/reports.jsonl`。
- DeepSeek prompt 禁止把任何情景价格改写成现在可预挂的期权订单；投递前校验所有数字和候选合约，缺失候选、改写数字或缺少“当前不可预挂”时直接回退原始模板。

## 剩余限制

触达时间模型目前仍是基于单日经验的 Brownian 缩放启发式，不应被视为校准后的 first-passage model。新的审计数据积累后，应按距离/EM、session、波动率环境和实际触达时长重新拟合，远墙在完成校准前只提供情景价，不提供静态执行承诺。
